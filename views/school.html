Pushers Game

Winter 2023 (ÉTS)

As part of the Data Structures and Algorithms course, giving a board game, we had the task to implement an AI that determines the best moves to play in that game. The game is called Pushers, where the player have to reach the opponent's last row by moving his pieces. The rules are simple:


  • Pushables are the smaller pieces and can only be pushed by pushers
  • Pushers are the bigger pieces and can move by themselves
  • Pieces can move up or diagonally, but not backward or sideways
  • Pieces can only capture diagonally

For this lab, we had to implement a move generator that returns all the possible moves, the minimax algorithm with pruning and the static evaluation function that determines the strategy of the AI.


Work time: 6 weeks

Skills:

  • Language(s): Java
  • Algorithms: MiniMax, Alpha-Beta Pruning
  • Data structures: 2D arrays
  • Tools: Git/GitHub, IntelliJ

Huffman algorithm

Winter 2023 (ÉTS)

As part of the Data Structures and Algorithms course, we had the task to implement the Huffman algorithm, which is used to compress and decompress files. This algorithm was implemented by creating the frequency table of each caracters in a file, building the Huffman tree from the frequency table, generating the Huffman code from the Huffman tree and encoding/decoding the file from the Huffman code.


Work time: 3 weeks

Final grade: 97.5%

Skills:

  • Language(s): Java
  • Algorithms: Huffman, Recursive
  • Data structures: Binary search tree, HashMap, ArrayList, PriorityQueue
  • Tools: Git/GitHub, IntelliJ

Chain of Production Simulation

Fall 2022 (ÉTS)

As part of the Object-Oriented Programming and Conception course, we were asked to program a chain of production that produce airplanes. In this simulation, different factories are needed to produce airplanes. For instance, metal factories are needed to create metal. The metal will be sent to wing and engine factories to create those components. Lastly, wings and engines are sent to the assembly factory, that produce the airplane. Produced airplanes are sent to a warehouse, where they will sell the airplanes (with fixed interval or at random). The warehouse also have a maximum capacity and have to stop the production when it is full. Before all, we were asked to think about the system's conception, by making a UML class diagram and sequence diagrams for important operations.


Work time: 4 weeks

Final grade: 91%

Skills:

  • Language(s): Java, XML, UML
  • Framework: JavaSwing
  • Pattern Designs: Observator, Stategy, Singleton
  • Tools: Git/GitHub, IntelliJ, Visual Paradigm
